# Philosophers
I never thought philosophy would be so deadly


🍝 Philosophers – 42 Project

Welcome to the world of forks, threads, and existential crises.

This project is inspired by the famous Dining Philosophers problem, where a group of philosophers sit around a table, endlessly cycling through three essential activities:

🧠 Thinking
🍝 Eating
😴 Sleeping

Sounds peaceful? Not quite.

⚔️ The Problem

Each philosopher needs two forks to eat… but there aren’t enough forks for everyone to grab them at the same time.

This leads to some classic chaos:

🍴 Philosophers fighting over forks
⏳ Waiting forever (deadlocks)
💀 Starving to death (literally)

The mission is to simulate this system while keeping everything running smoothly.

🧵 What’s Under the Hood

Behind the scenes, this project is all about:

Threads / Processes 👥
Mutexes & Semaphores 🔒
Precise timing ⏱️
Synchronization ⚙️

Build a system where multiple philosophers act independently… but don’t destroy each other in the process.

🎯 Objective

Create a simulation where:

Philosophers don’t die unexpectedly 💀
Forks are shared safely 🍴
Output is clean and not a total mess 🖨️
The system avoids deadlocks and starvation 🚫
🧠 Why This Matters

This isn’t just about philosophers.

It’s about understanding how real systems handle:

Multiple tasks running at the same time
Shared resources under pressure
Preventing everything from freezing or crashing

In other words: this is a first real step into concurrent programming.

🚀 Final Thought

In this project, timing is everything…
Take too long? 💀
Grab forks too aggressively? 🔒 Deadlock.
Don’t think things through? 😵 Chaos.

Words of wisdom: So code wisely — your philosophers’ lives depend on it.
